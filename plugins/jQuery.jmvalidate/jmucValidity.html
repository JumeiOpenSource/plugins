【events】（默认："change"）：除浏览器默认外要要进行表单验证的事件（多个事件空格分隔）

用户中心表单验证

基于$.h5validity实现
注意的要素
  1. 表单元素应有class为input_container的父元素
  2. 错误提示信息可用class: customError,patternMismatch,rangeOverflow,rangeUnderflow,stepMismatch,tooLong,typeMismatch,valueMissing。插件会根据错误类型控制其显示隐藏
  3. 表单错误状态可用class: .select_err,.textarea_err,.input_text_err,.input_password_err
  4. 正确提示信息class: valid，插件会在验证错误时将其隐藏，正确时重新显示

引用方式：

<script type="text/javascript" src="js/jquery.jmvalidate.js" data-h5f="js/H5F.js"></script>
注意在script标签写入data-h5f属性，以便在低端浏览器中按需加载

示例：

表单验证 
<label class="input_container">
	<span class="lab">用户名</span>
	<input type="text" required pattern="\w+">
	<span class="valueMissing">请填写您的用户名</span>
	<span class="patternMismatch">用户名应该为数字字母和下划线组成</span>
	<span class="valid">必填项，请输入数字字母和下划线作为用户名</span>
</label>

<label class="input_container">
	<span class="lab">密码</span>
	<input type="password" required>
	<span class="valueMissing">请输入密码</span>
	<span class="valid">必填项</span>
</label>

<label class="input_container">
	<span class="lab">再次输入密码</span>
	<input type="password" required>
	<span class="valueMissing">请再次输入密码</span>
	<span class="customError">两次密码输入不一致</span>
	<span class="valid">必填项， 两次输入必须一致</span>
</label>

调用方式：

$("#demo1").jmucvalidate();

//自定义验证，两次密码输入必须一致
var txtPwd = $("#demo1 :password").bind("change", function(){
	txtPwd[1].setCustomValidity(txtPwd.eq(0).val() == txtPwd.eq(1).val() ? "" : "两次密码输入不一致");
});
